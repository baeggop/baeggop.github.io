기본 정보

  • Base URL: /order
  • Authentication: Optional (Depending on your implementation)

목차

  • 1. GET /:tableId - 특정 테이블의 미결제 주문 조회
  • 2. POST /:tableId - 특정 테이블에 주문 추가
설명: 특정 테이블의 미결제 주문을 조회합니다.
Request
tableId: 테이블 ID (string, 필수)
Response
[
  {
    "id": "string",                 // 주문 ID
    "isPaid": "N",                  // 결제 여부 ('Y' 또는 'N')
    "orderDetails": [
      {
        "id": "string",             // 주문 상세 ID
        "menu": {
          "id": "string",           // 메뉴 ID
          "name": "string"          // 메뉴 이름
        },
        "menuOption": {
          "id": "string",           // 메뉴 옵션 ID
          "name": "string"          // 메뉴 옵션 이름
        },
        "menuOptionItem": {
          "id": "string",           // 메뉴 옵션 항목 ID
          "name": "string"          // 메뉴 옵션 항목 이름
        }
      }
    ]
  }
]
Error Responses
{
  "error": "Table not found"
}
{
  "error": "Failed to retrieve unpaid orders for the table"
}
설명: 특정 테이블에 새로운 주문을 추가하거나 기존 미결제 주문을 업데이트합니다.
Request
tableId: 테이블 ID (string, 필수)
Body
[
  {
    "menuId": "string",        // 메뉴 ID
    "quantity": 1              // 주문 수량
  }
]
Response
{
  "message": "Order saved successfully"
}
Error Responses
{
  "error": "Table not found"
}
{
  "error": "Menu ID and quantity are required"
}
{
  "error": "Failed to save order"
}
