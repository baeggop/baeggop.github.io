목차

  • 1. GET /:storeId - 특정 가게의 미결제 주문 조회
  • 2. GET /:tableId - 특정 테이블의 미결제 주문 조회
  • 3. POST /:tableId - 테이블에 새로운 주문 추가
설명: 특정 가게에서 미결제 상태인 모든 주문을 조회합니다.
Request
storeId: 가게 ID (string, 필수)
Response
[
  {
    "id": "string",
    "isPaid": "N",
    "table": {
      "id": "string",
      "name": "string"
    },
    "orderDetails": [
      {
        "id": "string",
        "menu": {
          "id": "string",
          "name": "string"
        }
      }
    ]
  }
]
Error Responses
{
  "error": "No active orders found for this store"
}
설명: 특정 테이블의 미결제 주문을 조회합니다.
Request
tableId: 테이블 ID (string, 필수)
Response
{
  "id": "string",
  "isPaid": "N",
  "orderDetails": [
    {
      "id": "string",
      "menu": {
        "id": "string",
        "name": "string"
      }
    }
  ]
}
Error Responses
{
  "error": "No active order found for this table"
}
설명: 특정 테이블에 주문을 추가합니다. 이 API는 테이블에 주문을 생성하거나, 이미 존재하는 미결제 주문의 세부사항을 업데이트합니다.
Request
tableId: 테이블 ID (string, 필수)
Body
[
  {
    "menuId": "string",
    "quantity": 1
  }
]
Response
{
  "message": "Order saved successfully"
}
Error Responses
{
  "error": "Table not found"
}
{
  "error": "Menu ID and quantity are required"
}
{
  "error": "Failed to save order"
}
